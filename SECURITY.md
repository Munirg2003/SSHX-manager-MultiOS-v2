# Security policy

If you discover a security vulnerability in this project, please follow these steps.

Reporting a vulnerability
1. Do not open a public issue.
2. Send an email to the repository owner: security@munirg2003.example (replace with the actual contact address).
3. Include:
   - A clear description of the vulnerability
   - Steps to reproduce
   - Impact and affected versions
   - Suggested mitigation (if any)

Response policy
- We will acknowledge receipt within 48 hours.
- We will triage and respond with a timeline for a fix.
- Fixed releases will include an advisory and recommended upgrade instructions.

Releases and signing
- We sign release artifacts and publish SHA256 hashes. Verify signatures and checksums before running any installer.
